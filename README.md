## Project Overview

This repository contains my Open Source Software Audit Project based on **Linux Process Scheduling**.

The purpose of this project is to understand how Linux manages multiple processes efficiently using scheduling algorithms. Instead of selecting a specific software like Python or MySQL, this project focuses on a **core operating system concept** — how the Linux scheduler ensures fairness and performance.

The study mainly explores the **Completely Fair Scheduler (CFS)**, which is used in modern Linux systems.

---

## Selected Topic

**Topic:** Linux Process Scheduling
**Category:** Operating System Concept
**Focus Area:** CPU Scheduling and Resource Management

Linux process scheduling is responsible for deciding which process gets CPU time. It plays a critical role in system performance, responsiveness, and multitasking.

---

## Project Objectives

The main objectives of this project are:

* To understand how process scheduling works in Linux
* To study different scheduling algorithms like FCFS, Round Robin, and CFS
* To analyze how fairness and performance are balanced
* To gain practical experience using Linux shell scripting
* To explore system-level behavior using scripts

## Repository Structure

This repository includes the following scripts:

---

## Script Descriptions

### Script 1 – System Identity Report

Displays basic system information such as:

* Kernel version
* Current user
* System uptime
* Date and time

---

### Script 2 – Process Scheduler Inspector

Checks CPU scheduling-related information:

* Displays running processes
* Shows CPU usage
* Explains scheduling concept

---

### Script 3 – Disk and Permission Auditor

Analyzes important system directories:

* Directory size
* Permissions
* Ownership

---

### Script 4 – Log File Analyzer

Analyzes system logs:

* Counts occurrences of keywords like "error"
* Displays recent log entries

---

### Script 5 – Open Source Manifesto Generator

Interactive script that:

* Takes user input
* Generates a philosophy statement
* Saves it to a file

## How to Run the Scripts

### Step 1: Make scripts executable

```bash
chmod +x *.sh
```

### Step 2: Run scripts

```bash
./script1_system_identity.sh
./script2_package_inspector.sh
./script3_disk_auditor.sh
./script4_log_analyzer.sh /var/log/syslog
./script5_manifesto_generator.sh
```
